1. An Overview of the AWS Edge Infrastructure Components [AWS: Infrastructure: AWS Edge Locations, AWS Regional Edge Cache, Edge]
    1. Note: The components below are better described as edge components, however they are full citizens of the AWS Global Infrastructure ecosystem. The scope of these components is not global. It is regional, like AWS Ground Station, or a data center facility scope, like AWS Outposts
    2. Edge Infrastructure Components (or AWS Infrastructure for the Edges)
      1. AWS Ground Station
        1. "AWS Ground Station is a fully managed service that lets you control satellite communications, process data, and scale your operations without having to worry about building or managing your own ground station infrastructure."
        2. "With AWS Ground Station, you have direct access to AWS services and the AWS Global Infrastructure including a low-latency global fiber network."
        3. "You can save up to 80% on the cost of your ground station operations by paying only for the actual antenna time used, and relying on the global footprint of ground stations to download data when and where you need it."
        4. "There are no long-term commitments."
        1. References:

      2. AWS Wavelength
        1. "AWS Wavelength is designed to deliver ultra-low latency applications to 5G devices by extending AWS infrastructure, services, APIs, and tools to 5G networks. Wavelength embeds storage and compute inside telco providers’ 5G networks to help developers build new applications for 5G end-users requiring single-digit millisecond latency, such as IoT devices, game streaming, autonomous vehicles, and live media production."
        2. "AWS Wavelength brings AWS services to the edge of the 5G network, minimizing latency when connecting to an application from a mobile device."
        3. "Wavelength Zones are AWS infrastructure deployments that embed AWS compute and storage services within communications service providers’ (CSP) datacenters at the edge of the 5G network, so application traffic can reach application servers running in Wavelength Zones without leaving the telecommunications network. This avoids the latency that would result from application traffic having to traverse multiple hops across the Internet to reach their destination, enabling customers to take full advantage of modern 5G networks."
        4. "Wavelength Zones allow developers to build applications that deliver ultra-low latencies to 5G devices and end users. Wavelength deploys standard AWS compute and storage services to the edge of telecommunication carriers' 5G networks."
        5. "A Wavelength Zone is a logical extension of a Region, and is managed by the control plane in the Region."
        6. "A Wavelength Zone is represented by a Region code followed by an identifier that indicates the Wavelength Zone, for example, us-east-1-wl1-bos-wlz-1."
        7. "Each Wavelength Zone is specific to a carrier and location. Wavelength Zones are connected to the parent Region through a redundant and high-bandwidth private network, which provides the applications running in Wavelength Zones fast, secure, and seamless access to the rest of AWS’ services."
        8. "Wavelength will be available within CSP networks such as Verizon, Vodafone, KDDI, and SK Telecom in places like the United States, Europe, Japan, and Korea to enable you to deliver ultra-low latency applications to your global base of end-users and connected devices."
        9. "AWS announced a partnership with Verizon in December 2019. Today Wavelength Zones are available through Verizon in several US cities, including: Chicago, Atlanta, Boston, Dallas, Denver, Las Vegas, Miami, New York City, San Francisco, Seattle, and Washington DC.
          Other partnerships include Vodafone, for service in London, UK; KDDI for service in Tokyo and Osaka, Japan; SK Telecom, for service in Daejeon, South Korea."
        10. The CSP does not need to buy and place hardware in their data center. Hardware and services are provided by AWS. With a Carrier Gateway for connectivity between the Region and the CSP data center.
        11. Services available in the AWS Wavelength Zone includes, Amazon EC2 instances, Amazon EBS volumes, and Amazon VPC subnets and carrier gateways. "You can also use services that orchestrate or work with EC2, EBS and VPC such as Amazon EC2 Auto Scaling, Amazon EKS clusters, Amazon ECS clusters, Amazon EC2 Systems Manager, Amazon CloudWatch, AWS CloudTrail, and AWS CloudFormation. The services in Wavelength are part of a VPC that is connected over a reliable, high-bandwidth connection to an AWS Region for easy access to services including Amazon DynamoDB and Amazon RDS"
        12. Use cases for AWS Wavelength includes: Connected vehicles, Interactive live video streams, AR/VR, ML-assisted diagnostics for healthcare, Real-time gaming, e.g. "with AWS Wavelength, the most demanding games can be made available on end-user devices that have limited processing power by streaming these games from game servers in Wavelength Zones."

      1. AWS Outposts
        1. "AWS Outposts is designed for workloads that must remain on-premises due to latency requirements, but where customers want that on-prem workload to run seamlessly with their AWS workloads. AWS Outposts are fully managed and configurable compute and storage racks built with AWS-designed hardware, which allows customers to run compute and storage on-premises while seamlessly connecting to AWS’s broad array of cloud services."
        2. "AWS Outposts brings native AWS services, infrastructure, and operating models to virtually any data center, co-location space, or on-premises facility."
        3. Extends the AWS infrastructure and services, onto a customer's on-premises or co-location facility
        4. "AWS Outposts enables native AWS services, infrastructure, and operating models in on-premises facilities. In AWS Outposts environments, you can use the same AWS APIs, tools, and infrastructure that you use in the AWS Cloud."
        5. It is made available 42U racks or smaller form-factors
        6. "Once the rack is installed on-site, the customer is responsible for power, connectivity into the LAN, and environmental prerequisites such as temperature, humidity, and airflow.
          The customer is also responsible for the physical security of the Outpost location, however each rack has a lockable door and tamper detection features.
          Each server is protected by a removable and destroyable Nitro hardware security key.
          Data on the Outpost is encrypted both at-rest, and in-transit between nodes in the Outpost and back to the AWS region."
        7. There are a growing number of AWS services: compute, storage, database, and other services, available via AWS Outposts
        8. It is an IT-as-a-Service platform from Amazon Web Services (AWS), deployed at the customer's facility
        9. It provides a connection path back to the AWS Region
        10. Customers use AWS Outposts to run services in their local environments, like Amazon Elastic Compute Cloud (EC2), Amazon Elastic Block Store (EBS), and Amazon Relational Database Service (RDS), and are ideal for workloads that require low latency access to on-premises systems, local data processing, or local data storage.
        11. Other challenges that could be addressed by using AWS Outposts include a need to meet data residency requirements, which demands that certain data or applications reside locally, and legacy software that are not cloud-ready.
        12. AWS Outposts is similar to AWS Wavelength in a number of ways, however it addresses different use cases.

      2. AWS Snow Family
        1. A family of appliances for data transfer or data processing at the edge.
        2. AWS Snowball is a service of ruggedized physical storage and computing devices, that enable you to move data in and out of AWS.
        3. The service helps overcome challenges that you can encounter with large-scale data transfers, including long transfer times, insufficient bandwidth, and restricted access to the database from outside the domain.
        4. AWS Snowball Edge is an AWS service you can use to transfer data to the cloud at faster-than-network speeds using an AWS-owned appliance.
          An AWS Snowball Edge device can hold up to 80 TB of usable storage. It uses 256-bit encryption and an industry-standard Trusted Platform Module (TPM) to ensure both security and full chain-of-custody for your data.
        5. Appliance form factors:
          1. Snowcone
            • Use case: IoT, Edge processing
            • Usable capacity: 8 TB per appliance
          2. Snowball Edge
            • Use case: data transfer, edge processing
            • Three versions:
              • Compute-optimized
                • 42 TB (39.5 usable) HDD capacity
                • 52 vCPUs
                • 7.68 TB of dedicated NVMe SSD
              • Compute Optimized with GPU
                • 42 TB (39.5 usable) HDD capacity
                • 52 vCPUs, GPU
                • 7.68 TB of dedicated NVMe SSD
              • Storage-optimized
                • 80 TB usable HDD capacity
                • 40 vCPUs
                • 1 TB of additional SSD storage
            • Depending on the country and the shipping plan, it usually takes anywhere from 1 to 7 working days to deliver the snow devices
          3. Snowmobile
            • Use case: data transfer
            • Usable capacity: 100 PB
            • Shipping container worth of storage capacity
        6. Workflow:
          1. Customer orders device from AWS
          2. Device shipped to customer (or driven for Snowmobile)
          3. Customer connects device to network, transfers files to it
          4. Customer ships appliance to AWS
          5. AWS downloads appliance data to S3 bucket
          6. AWS gives customer access to S3 bucket

      3. AWS Storage Gateway
        1. A storage appliance that provides a storage conduit between AWS and customer data center
        2. Gateway types:
          1. Amazon S3 File Gateway
            • Supported Protocol: NFS, SMB
            • "Amazon S3 File Gateway presents a file interface that enables you to store files as objects in Amazon S3 using the industry-standard NFS and SMB file protocols, and access those files via NFS and SMB from your data center or Amazon EC2, or access those files as objects directly in Amazon S3"
          2. Amazon FSx File Gateway
            • Supported Protocol: SMB
            • "Amazon FSx File Gateway provides fast, low-latency on-premises access to fully managed, highly reliable, and scalable file shares in the cloud using the industry-standard SMB protocol."
          3. Volume Gateway
            • Supported Protocol: iSCSI
            • "Volume Gateway presents your applications block storage volumes using the iSCSI protocol."
          4. Amazon Tape Gateway
            • Supported Protocol: iSCSI VTL
            • "Tape Gateway presents a virtual tape library (VTL) consisting of virtual tape drives and a virtual media changer to your backup application using storage industry standard iSCSI protocol."
        3. The AWS Storage Gateway can be deployed in either a virtual or a physical appliance format:
          1. virtual appliance with: VMware ESXi, Microsoft Hyper-V, or Linux KVM
          2. hardware appliance
          3. virtual machine in VMware Cloud on AWS
          4. or as an Amazon Machine Image (AMI) in Amazon EC2

    1. Here are several descriptions/definitions of an AWS Region to help enforce the concept:
      1. An AWS Region is a geographical location around the world where a complement of AWS Services is accessible, across multiple Availability Zones (AZ).
      2. Each AWS Region consists of multiple, isolated, and physically separate AZ's within a geographic area. Each AZ within the Region consists of one or more physical data centers.
      3. AÂ  Region spans a large geographic area, e.g. a metropolitan area. It is an area where AWS has a presence, and deploys its services in a highly-available manner, using multiple Availability Zones
      4. Each AWS Region exists wholly within a single country domain
      1. "AWS Regions include multiple Availability Zones. Each Availability Zone consists of discrete data centers with redundant power and networking. They are housed in separate facilities, in locations that are carefully selected to mitigate environmental risks."
      1. Customers can take advantage of the multiple Availability Zone infrastructure to deploy applications and databases that are highly-available.
      2. Customers can avoid having a single point of failure in their architecture by designing their applications to run in multiple Availability Zones in the same Region for high-availability and across multiple Regions to achieve a degree of fault-tolerance.
        1. Note: While it is possible to design an architecture that is active/active across multiple Regions, i.e. use multiple Regions to meet their availability goals for an "always-on" service, and there are customers who do this, e.g. Netflix, most customers who use multiple Regions do so for backup or disaster recovery purposes, or to meet data governance laws.
      3. Synchronous data replication across different Availability Zone in the same Region or asynchronous replication across Regions is possible.
      4. As of July 2022, every standard AWS Region has three or more Availability Zones (AZ). However the formal description is that an AWS Region consists multiple AZs, i.e. two or more AZs.
        1. Note: AWS may limit the number of AZs visible to new AWS customers in some Regions, e.g. New customers can access two Availability Zones in US West (Northern California). As per this premium support document.
