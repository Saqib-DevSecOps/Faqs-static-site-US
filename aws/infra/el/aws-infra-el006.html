    2. It doesn't necessarily know
    3. When you create a distribution, CloudFront assigns a domain name to the distribution, such as d111111abcdef8.cloudfront.net. You can use this domain name in the URLs for your content, e.g. d111111abcdef8.cloudfront.net/index.html.
    4. The end-user accesses CloudFront via the CloudFront Distribution domain name. Alternatively CloudFront can be setup to use your own domain name, e.g. www.example.com/index.html. This name must be in DNS, via Amazon Route53.
    5. A viewer, or client makes a request for the Origin server (website or application), requesting an object. The request is sent to DNS, which responds with the nearest CloudFront POP (Edge Location), based on latency.
    6. From there, CloudFront protocols take over to serve the content to the client in the most performant manner. See the How CloudFront delivers content link below.
