1. Can CloudFront be used to distribute a folder, or does it operate only at the bucket level? [AWS: Infrastructure: CloudFront]
    1. TL;DR - Amazon CloudFront can be used to distribute content from Amazon S3. The Origin, or the definitive source of the content CloudFront distributes CAN be from an Amazon S3 Bucket OR from a folder in an S3 Bucket.
    1. Background:
      1. A CloudFront Origin is the source location of the content served via CloudFront. It is either an Amazon S3 bucket or a custom origin such as EC2 or an on-premises server
      2. To make content available via CloudFront, you would create a CloudFront distribution, which describes the what and how of making content available via CloudFront.
      3. In Amazon S3, unlike a traditional file system, folders (or directories) do not exist inherently. Files are not organized in hierarchies.
      4. In Amazon S3, buckets and objects are the primary resources, and objects are stored in buckets.
      5. Unlike a traditional file system, such as Amazon EFS, or Microsoft NTFS, Amazon S3 has a flat structure instead of a hierarchy.
      6. However, for the sake of organizational simplicity, the Amazon S3 console supports the folder concept as a means of grouping objects.
      7. If the Origin is S3, a CloudFront distribution will distribute any Object in the S3 Bucket, and since a "folder" in S3 is simply a type of S3 Object, it will be distributed.
