1. What's the difference between AWS Organizations OU vs IAM Group? [AWS: Management: AWS Organizations, Organizational Unit]
    1. An IAM group (or IAM user group) is a collection of IAM users. It is a container of one or more IAM users and can specify permissions for those IAM users.
    2. An IAM user is created by the AWS Root Account, or another IAM user. An IAM user can be given privileges after the fact, however it starts out with no privileges.
    3. An IAM policy, which grants and denies permissions, can be applied to an IAM group to control what the IAM users in the group can do, and what resources (e.g. Amazon EC2) they can access.
    4. An AWS Organizational Unit (OU) is a container of accounts designed to meet your budgetary, security, or compliance needs.
    5. At a very basic level it is similar to an IAM group, i.e. a collection of accounts.
    6. However there are important differences. An IAM group is a collection of IAM user accounts. An Organizational Unit is a collection of AWS Root Accounts. An IAM policy can be applied to an IAM group to control what IAM users can do. However an IAM policy cannot be applied to an AWS Account Root.
    7. The benefit of an Organizational Unit (as part of the AWS Organizations service) includes:
      1. An ability to implement a multi-account strategy, which is a best practice
      2. You can group like-account together and manage them in a seamless manner, e.g. accounts with the same budgetary, security, or compliance needs can be placed within the same OU and managed as a unit
      3. It provides the business an ability to group like accounts together, and manage them in a similar manner
      4. It is the only way to place restrictions on what the AWS Root Account can do.
