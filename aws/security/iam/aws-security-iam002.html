1. What is Programmatic Access? [AWS: Security: IAM, Credentials]
    1. Programmatic access is the use of an Access Key, to access AWS services
      1. An Access Key is type of AWS credential
      2. There are two types of AWS credentials used for accessing your account: passwords and access keys
      3. Passwords are used to sign-in to the AWS Management Console or Access Portal:
        • You can sign into the Console either as a Root User, with an email address and password, or an IAM User, with an account ID (or alias), user name, and password
        • To sign in through the AWS access portal, as an IAM Identity Center (formerly AWS Single Sign-On) user, you need a sign-in URL, user name, and password.
      4. Access Keys are a way to verify your identity, to authenticate yourself to AWS, in order to invoke actions in AWS programmatically:
        • Programmatic access could be either via Software Development Kits (SDKs), or the AWS Command Line Interface (AWS CLI)
      5. Both the AWS Root User and individual IAM Users can be assigned a password and/or access keys
      1. Note:
        2. It is a best practice NOT to use the AWS Root User account to perform daily tasks, and in particularly, do not access AWS programmatically as the Root User. As such, do not create Access Keys for the Root User, or if you do, lock the keys away securely.
    1. An Access Key is composed of an Access Key ID and a Secret Access Key
      1. Access Key ID (e.g. AKIAIOSFODNN7EXAMPLE)
      2. Secret Access Key (e.g. wA9SR7ZnFEMI+w7MDENGBAtSGCYEXAMP2jlZQXlK)
