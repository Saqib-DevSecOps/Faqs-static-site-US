1. Instance Types vs. Instance Family, are they the same thing? [AWS: Compute: Instances]
    1. Not the same thing
    2. In AWS Compute services, virtual machines are called instances, or EC2 Instances.
    3. AWS EC2 provides wide selection of Instance Types (or virtual hardware) with different configurations of CPU, memory, storage, and networking resources so a user can tailor their compute resources to the needs of their application.
    4. Various Instance Types are grouped into Instance Families with optimized to fit particular use cases
    5. EC2 Instances can be understood by breaking them down into:
      1. Instance Category
        1. EC2 Instance Categories are Instances grouped by capabilities and use-case.
        2. There are five Instance Categories:
          1. General purpose
          2. Compute optimized
          3. Memory optimized
          4. Storage optimized
          5. Accelerated Computing
        3. You can choose from one of the Instance Categories based on high-level use-case, e.g. the Compute Optimized Category for workloads that are compute-bound, needing access to high-performance CPUs.
        4. Note: Instance Category is my name for this group of five EC2 Instance types. Describing them as Instance Types, as in some documentation, is confusing. E.g.
          1. An Overview of Amazon EC2 and Amazon RDS Instance Families describes these as Instance Families
          2. Where Hardware Specifications describes Instance Families as we do below.
          3. Let me know if you find a more authoritative definition.
      2. Instance Family
        1. Within each Instance Category, there are multiple instance families. These allow you to further target the use-case to the class of instance, e.g. by processor capability, generation, etc.

          1.  Instance Category

             Instance Family

             General Purpose:

             Mac, T, M, A

             Compute Optimized:

             C, Hpc

             Memory Optimized:

             R, X, High Memory, z

             Accelerated Computing:

             P, DL, Trn, Inf, G, F, VT

             Storage Optimized:

             Im, Is, I, D, H
      3. Instance Generation
        1. After the Instance Family: the Instance Generation
        2. Each Instance Family is further classified by its Generation. Each subsequent release of an Instance Family is a newer generation.
        3. Instance Generation is an integer value, e.g. 1, 2, 3, 4, etc.
        4. M6 indicated the 6th generation of the M instance Type
        5. The newer the Instance Type, the higher the Instance Generation value, e.g. C7 is one generation newer than C6.
        6. Some older generation Instances are still available for use.
        7. Generally newer generation Instances are offered at the same or lower price point, even as they come with enhanced capabilities.
        8. AWS continues to support specific older generation Instance Types, though newer generation Instance Types are recommended to get the best performance for your workload.
      4. Instance Size
        1. TBA:
        1. M6.xlarge, the relative capacities for CPU, Memory and Network
        2. Each Instance has a size component. The Size indicates a relative combination of CPU, Memory and Network capacities.
          1. The Size value with the string, metal , represents a bare metal Instance that AWS can provision for customers.
      5. Instance Type naming convention can indicate additional capabilities, such as:

        1. a

          AMD EPYC processors

          g

          AWS Graviton processors

          i

          3rd gen Intel Xeon processor

          d

          Locally attached NVMe SSD

          n

          Network optimization

          b

          Block storage optimization

          e

          Extra storage or memory

          z

          High frequency

          en

          Dense NVMe SSD and network

          Inf

          AWS Inferentia chips

          metal

          Bare metal instance
    6. First select an EC2 Instance Family based on its general fit to your workload
      1. e.g. for workloads that are compute-bound and require high-performance CPUs, use Instance Types from the Compute Instance Family
    7. Next, from within the Instance Family, select an Instance Type
      1. A1 Instance Types deliver significant cost savings and are ideally suited for scale-out and Arm-based workloads.
