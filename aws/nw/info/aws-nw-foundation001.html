1. How do you create a network in AWS?
    2. [AWS: Networking: Foundations]
    1. The Amazon VPC or Virtual Private Cloud is a service that lets customers provision a logically isolated section of the Amazon Web Services (AWS) cloud, and then be able to launch AWS resources in that space.
    2. The Amazon VPC gives customers complete control over their virtual networking environment, including:
      1. selecting the IP address ranges, or CIDR-blocks
      2. creation of subnets
      3. configuration of route tables
      4. attaching network gateways
      5. deployment of hardware Virtual Private Network (VPN) to facilitate the connection between the corporate datacenter and the VPC.
    3. The characteristics of a Virtual Private Cloud include:
      1. When created, VPCs span all Availability Zones (AZs) within a single AWS Region
      2. Each AWS account can create one or more VPCs per Region
      3. VPCs can have the following components:
        1. Subnets - partitions the VPC into one or more virtual sub-networks
        2. Gateways - attach to the VPC to enable controlled access to and from the VPC; including Internet Gateway, Virtual Private Gateway (VGW), Network Address Translation Gateway (NAT), Egress-only Internet Gateway.
      4. VPCs are isolated by default on creation, access is enabled through the gateways, VPC Endpoints, and VPC Peering.
