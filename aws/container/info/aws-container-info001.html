1. Why is it recommended to only have one process per container? [AWS: Container: Technology]
    1. Note: Docker changed their language on this from process to concern.
    2. "It is generally recommended that you separate areas of concern by using one service per container."
    3. I.e. by recommendation, a container should have one main process.
    4. That process may fork into multiple processes. An example is where the Apache web server main process would start multiple worker processes to handle incoming HTTP requests.
    5. If you have a need to connect multiple processes together in a solution, you can tie multiple containers together using user-defined networks and shared volumes.
