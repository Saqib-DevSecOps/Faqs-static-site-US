1. Does AWS have support for cross Public Cloud integratio ? like Red Hat OpenShift? [AWS: Container: Amazon Elastic Kubernetes Service]
    1. TL;DR
      1. Take a look at Amazon Elastic Kubernetes Service (Amazon EKS) and Amazon Elastic Container Service (Amazon ECS), and more specifically, EKS Anywhere and ECS Anywhere.
      2. They enable AWS customers to create and operate Kubernetes clusters in a non-AWS environment, providing functionality similar to that of RedHat OpenShift on AWS (ROSA)
    2. RedHat OpenShift Container Platform is a Container Orchestration service from RedHat
      1. It is supported on AWS
      2. It is a cloud-based container platform that leverages the Kubernetes technology.
      3. It makes it easy to scale a container architecture to thousands of nodes serving millions of clients.
    3. "Kubernetes is open source software for automating deployment, scaling, and management of containerized applications."
      1. It deploys and manages containerized applications at scale.
      2. "Kubernetes arranges containers into logical groupings for management and discoverability, and in AWS, launches them onto clusters of Amazon Elastic Compute Cloud (Amazon EC2) instances."
      3. "Using Kubernetes, you can run containerized applications including microservices, batch processing workers, and platforms as a service (PaaS) using the same toolset on premises and in the cloud."
      4. Kubernetes comes with native features such as: Autoscaling, Load balancing, DNS management, etc. that make it capable of handling complex architecture requirements without needing to bring in external services. However, Kubernetes is known to be complex to setup and manage.
      5. Kubernetes is inherently vendor/platform agnostic. Making it a convenient technology for implementing hybrid and multi-cloud strategies.
    4. AWS' Container Orchestration services can be deployed in a hybrid cloud or multi-cloud architecture
    5. Amazon Elastic Kubernetes Service (Amazon EKS) is a managed service that makes it easy to run Kubernetes on AWS without installing and operating your own Kubernetes control plane or worker nodes.
      1. Amazon EKS enables customers to use Kubernetes without the undifferentiated heavy-lifting of self-managing the entire infrastructure.
      2. EKS provides a fully-managed Kubernetes control plane. It performs Container Orchestration by managing and distributing Containers across a cluster of nodes (EC2 Instances) on AWS
      3. Other similar services on AWS include Amazon Elastic Container Service (Amazon ECS) and Red Hat OpenShift on AWS (ROSA)
    6. As of 2021, Amazon EKS Anywhere, a deployment option for Amazon EKS that enables you to easily create and operate Kubernetes clusters in a non-AWS environment.
      1. "Amazon EKS Anywhere makes it possible for users to extend the container infrastructure outside the AWS cloud environment."
      2. Amazon EKS Anywhere provides a management layer and automation tooling, enabling an easier cluster operation, lifecycle management, observability, backups and policy management.
      3. Unlike Amazon EKS with a control plane managed by AWS, since Amazon EKS Anywhere runs on-premises, the control plane of Kubernetes is expected to be managed by the customer.
      4. "In addition, you can leverage the EKS console to view all of your Kubernetes clusters (including EKS Anywhere clusters) running anywhere, through the EKS Connector."
    7. EKS Distro is the same Kubernetes distribution used in AWS. It provides a standard, secure, and community supported version of Kubernetes. It is fully compatible with any open source cluster management tooling.
