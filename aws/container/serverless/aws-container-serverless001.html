1. What's the difference between Serverless and Fully-managed services? [AWS: Container: Serverless, Fully-managed]
    1. What was your honest, initial reaction when you first heard the term Serverless computing? Like most, including myself, you likely thought, "that's not possible, is it?"
    2. With both fully-managed and serverless, servers are involved.
    3. Serverless is a cloud-native development model. It allows developers to build and run applications without thinking about servers. With serverless computing, your application still runs on servers, however the server infrastructure is completely managed by AWS.
    4. The Serverless model masks (or abstracts) the details of the server infrastructure from the subscribers.
    5. Both Fully-managed and Serverless off-load the undifferentiated heavy-lifting from the customer.
      1. I.e. the tedious, often manual, yet impactful tasks, that typically doesn't involve access to the company's intellectual property. Infrastructure related tasks such as architecting, scaling configuring, provisioning, patching, securing, load balancing, sharding, high-availability, etc.
      2. These undifferentiated heavy-lifting tasks divert resources away from a business focus.
    6. Fully-managed and Serverless sit on a spectrum of how much input the cloud customer has on the server infrastructure of the services they consume.
      1. With serverless computing, and in spite of the name, serverless computing involves the use of servers. It's just that the customer has no input on how the server infrastructure is configured, and after the infrastructure is deployed, the customer has no access to it. Also it is the computing model where the cloud provider is responsible for both for managing the cloud infrastructure and the scaling of applications.
      2. Fully-managed service seeks customer input on configuration of the server infrastructure, and depending on the service, may or may not have access to the server infrastructure, after deployment.
    7. One key difference between the two is, with Fully-managed some input from the customer on how the server environment is setup, is expected and sought. With Serverless, no input from the customer is expected or sought.
    8. With Serverless, the cloud provider fully manages server infrastructure all the way up to the application layer. Fully-managed generally stops short of the application layer.
