1. When should I use Amazon ECS or Amazon EKS without AWS Fargate?
    2. [AWS: Container: Orchestration]
    1. When you want more control
    1. Background
      1. AWS Fargate is a serverless compute engine for containers. It works with both Amazon Elastic Container Service (ECS) and Amazon Elastic Kubernetes Service (EKS).
      2. Fargate eliminates the need to provision, patch, or manage cluster capacity for containerization. Making it easy to focus on building your applications
      3. With Fargate, you don't pay for idle compute capacity.
        1. Pricing is based on requested vCPU, memory, Operating Systems, CPU Architecture, and storage resources
        2. Fargate has the advantage of allowing you to pay only for the capacity (vCPU, memory, and storage) consumed by your containerized applications
    2. Answer
      1. Use ECS or EKS without Fargate, if you need greater control, e.g. in terms of configurations and broader customization options,
        1. Fargate does not support the full set of vCPU and Memory options available with Amazon ECS and Amazon EKS.
          With Fargate you choose from a supported configuration of vCPU options and a corresponding list of memory amounts. See Supported Configurations
        2. Also, GPUs are not currently supported on Fargate. Use Fargate for workloads that run on CPUs
