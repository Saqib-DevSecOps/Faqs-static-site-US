1. What is an AWS Account? [AWS: General: AWS Account, General]
    1. AWS accounts are a fundamental part of accessing AWS services. You can think of an AWS account in three basic ways. As a:
      1. Container
        1. Think of your AWS account as a container for all your AWS resources (Elastic Compute Cloud, S3 bucket, IAM user, etc.), and the AWS identities that can access those resources
        2. It contains
          1. a set of AWS resources enabled in the account
          2. a set of users including the Account Root user and zero or more IAM users
            1. the Root user comes pre-created with the AWS Account
            2. The root user can optionally create one or more IAM users
          3. settings used to configure the account and its resources
        3. Note: Every resource is uniquely identified by an Amazon Resource Name (ARN) that includes the account ID of the account that contains, or owns it
      2. Security boundary
        1. AWS accounts serve as the fundamental security boundary for your AWS resources. Enabling a level of isolation and access control.
        2. Resources that you create in your account are available to users who have credentials for your account.
      3. User or set of users
        1. There are two general user types in AWS:
          1. AWS Account Root User
          2. Amazon IAM User
        2. When you sign-up for AWS service, you are provisioned an AWS account, and that comes with an AWS account root user
        3. Every AWS account, i.e. the Container, can have one AWS Account Root User, and zero or more IAM Users
        4. To access the AWS Account for the first time, login using provided AWS account root user identity
        5. Optionally create one or more IAM users
