1. What is the Amazon Resource Names (ARN)?
    2. [AWS: General: Information, Services]
    1. Amazon Resource Names (ARNs) uniquely identify AWS resources.
    2. Customers and developers can use ARNs to specify a resource unambiguously across AWS, including use in IAM policies, Amazon Relational Database Service (Amazon RDS) tags, and API calls.
    3. An ARN can be specified in one of the following three formats:
      4.  arn:partition:service:region:account-id:resource-id 
       arn:partition:service:region:account-id:resource-type:resource-id 
       arn:partition:service:region:account-id:resource-type/resource-id 
    1. Each ARN is composed of several fields:
      1. ARN -- fixed field with the name arn
      2. Partition -- A partition is a group of AWS Regions, identified by the following partition names. All resources in an AWS account belong to one of these partitions:
        1. aws - AWS Regions -- Commercial. Most AWS accounts are using the public partition.
        2. aws-cn - China Regions -- China commercial. Used by accounts tied to entities in China.
        3. aws-us-gov - AWS GovCloud (US) Regions. Used by accounts tied to US government public sector entities
      3. For completeness, there are partitions defined for the secret and top-secret regions:
        1. aws-iso - Used by the U.S. Intelligence Community (IC) to operate workloads up to the Secret U.S. security classification level.Â 
        2. aws-iso-b - Used by the U.S. Intelligence Community (IC) to operate workloads up to the Top Secret U.S. security classification level. The highest security classification in the country.
      4. Service -- The service namespace that identifies the AWS product. I.e. the name of the service the resource belongs to, e.g.
        1. s3 for Amazon S3
        2. ec2 for Amazon Elastic Compute Cloud
      5. Region -- The Region code. I.e. the Region the resource is located in. E.g. us-east-2 for US East (Ohio).
      6. Account-ID -- The AWS account identifier that owns the resource. E.g., 123456789012.
      7. Resource-ID -- The resource identifier. This part of the ARN can be the name or ID of the resource or a resource path. For example, user/Bob for an IAM user or instance/i-1234567890abcdef0 for an EC2 instance.
