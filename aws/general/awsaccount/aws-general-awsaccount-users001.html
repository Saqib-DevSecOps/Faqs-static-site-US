1. What are the different user accounts in AWS? [AWS: General: AWS Account, Users]
    1. The root user (also called account owner or account root user), is the default user that comes with an AWS account. It has full, unrestricted privileges in the AWS account. As such, it should not be used for day-to-day tasks.
      1. The root user is created when the AWS account is created
      2. The credentials of the account owner allow full access to all resources in the account
      3. The root user can sign in to the AWS Management Console using an email address and password
    2. The IAM user is a user account created by the root user. IAM users can be assigned privileges/permissions.
      1. IAM users are created by the root user or a privileged IAM user.
      2. The IAM user can sign in to the AWS Management Console using the account alias or 12-digit AWS account ID, the IAM user name, and the IAM user's password
    3. By default, a new IAM user has no privileges. However it can be assigned privileges. An IAM user granted administrator privileges is referred to as an IAM administrator.
    4. For day-to-day administrative tasks, create one or more IAM users and assign them appropriate privileges. For example, you can create an IAM user with appropriate privileges to manage Network Access Control Lists, instead of using the root user.
